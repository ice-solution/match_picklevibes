PickleVibes 賽事報名／預約（match_picklevibes）

本文件記錄專案功能、HTTP 路由、資料模型與主要函式，供後續開發對照。內容依目前程式庫整理；實際行為以 server.js 及各模組為準。

1. 專案定位

Node.js + Express + MongoDB（Mongoose）+ EJS 的雙人組報名網站：訪客填寫聯絡人與兩位球員資料、選組別、同意私隱聲明後送出；系統寫入資料庫、可寄送登記完成電郵。管理員可於後台檢視名單、匯出 Excel、透過 Stripe Checkout 收取報名費，並以 Webhook 更新付款狀態。

產品語意上可稱「比賽預約／登記」；程式層為 Registration（報名） + Payment（Stripe），並無獨立「時段預約」模組。

2. 技術棧與指令

  • 執行：node server.js（npm start）、開發 npm run dev（nodemon）
  • 樣式：Tailwind（npm run build:css / watch:css）
  • 依賴：express、mongoose、ejs、express-session、stripe、nodemailer、xlsx、dotenv

3. 使用者（訪客）流程

  • 首頁 GET /：賽事資訊、報名是否開放（REGISTRATION_DEADLINE）、可選 WhatsApp 連結
  • 報名表 GET/POST /apply：截止後回傳「已截止」頁（pages/closed）
  • 成功頁 GET /success?rid=：顯示剛建立的報名摘要
  • 長效付款連結 GET /pay/:token：依 token 找報名，若未付款則即時建立 Stripe Checkout Session 並 303 轉跳（避免舊 session URL 約 24 小時失效）
  • 付款結果頁：GET /payment/success、GET /payment/cancel

5. 資料模型（Mongoose）

Registration（models/Registration.js）

  • 聯絡人：fullName、email、phone、bocReferralCode
  • 兩位球員：player1 / player2（name、dateOfBirth、gender、duprNR、kidNoDuprScore、dupr）
  • 賽事快照：division、tournamentName、tournamentDate、tournamentLocation、notes
  • 同意書：consentAccepted、consentAcceptedAt
  • 電郵：emailSentAt、emailSendError
  • 付款：paymentStatus（unpaid | pending | paid）、paidAt、latestStripeCheckoutSessionId、latestPaymentAmountCents、paymentLinkToken、paymentLinkTokenCreatedAt
  • 唯一索引：email + tournamentName + division（同一電郵同一賽事同一組別不可重複報名）

PaymentTransaction（models/PaymentTransaction.js）

  • registration、stripeCheckoutSessionId（唯一）、stripePaymentIntentId、amountCents、currency
  • status：pending | paid | expired | failed；checkoutUrl、paidAt
  • 付款電郵追蹤：paymentEmailSentAt、paymentEmailSendError、lastWebhookEventId

6. 核心模組與匯出函式

lib/divisions.js

  • DIVISIONS：組別選項 value / label 列表
  • divisionLabel(value)、divisionLabelOrValue(value)
  • isDivisionExemptFromRules(value)：親子組等豁免規則檢查
  • divisionFeeCents(value)：依組別回傳 Stripe 金額（仙）；小朋友組用 REGISTRATION_FEE_KIDS_CENTS，其餘 REGISTRATION_FEE_DEFAULT_CENTS

lib/divisionRules.js

  • GENDER、isParentChildDivision、isKidsDivision、ageOnDate(dob, ref)
  • parseDuprInput(raw)：DUPR 2.00–4.00、小數兩位
  • allowNRForDivision(division)：公開組、3.0-、親子組可 NR
  • validateDivisionPlayers(division, p1, p2, refDate)：年齡、DUPR 上下限、雙打性別組合與 RULES 對照
  • RULES：各組別規則對照表

lib/email.js（匯出）

  • duprOrNR(player)：顯示用 DUPR / NR / 沒有 DUPR 積分
  • sendRegistrationEmail({ tournament, reg })
  • sendPaymentRequestEmail({ tournament, reg, checkoutUrl, amountCents, publicBaseUrl })
  • sendPaymentReminderEmail({ tournament, reg, payLinkUrl, amountCents, publicBaseUrl })

server.js 內部重要函式（非路由，供維護時搜尋）

  • 賽事／時間：getAgeReferenceDate()、getRegistrationDeadline()、isRegistrationOpen()
  • Stripe：stripeClient()、stripeWebhookSecret()、stripeConfigured()、computeCheckoutExpiresAtUnixSec()
  • 網址：baseUrl(req)、appPublicBaseUrl(req)
  • 付款連結：randomToken()、ensurePaymentLinkToken(regId)、paymentLinkUrlForRegistration(regId, req)
  • Checkout：createCheckoutSessionForRegistration(regLean, req)、sendCheckoutPaymentEmailToRegistrant(...)、createCheckoutSessionAndSendPaymentEmail(regLean, req)
  • 寄信：sendPaymentLinkEmailToRegistrant、sendPaymentReminderEmailToRegistrant
  • 報名表：normalizeEmail、isValidEmail、defaultApplyValues、renderApply / renderApplyWithStats
  • 管理：requireAdmin、adminConfigured

7. 業務規則摘要

  • 報名表欄位驗證：聯絡人、電郵格式、電話、組別、私隱同意；可選推廣碼若填寫須等於 BOC_EXPECTED_REFERRAL_CODE（預設 BOCLP26）
  • 親子組：不套用 validateDivisionPlayers 的年齡／DUPR／性別規則
  • 小朋友組（U8–U14）：不可 NR；可勾「沒有 DUPR 積分」；年齡為「該歲或以下」
  • 30+ / 50+：依參考日計算年齡；男雙／女雙／混雙性別組合需符合組別
  • DUPR 3.0- / 3.0+：與輸入數值交叉驗證；3.0+ 不可 NR

8. Stripe 與 Webhook

  • Checkout 建立時可把 PAYMENT_WINDOW_END_AT 轉成 expires_at（受 Stripe 最長約 24 小時限制，程式會取較早者）
  • Webhook 成功付款：更新 PaymentTransaction 為 paid，並將對應 Registration 設為 paid
  • Session 過期：交易狀態改為 expired

9. 環境變數

完整範例見專案根目錄 .env.example。常用項包括：MONGODB_URI、PORT、TOURNAMENT_*、TOURNAMENT_AGE_REFERENCE_DATE、REGISTRATION_DEADLINE、APP_BASE_URL、STRIPE_*、REGISTRATION_FEE_*_CENTS、ADMIN_USER / ADMIN_PASS、SESSION_SECRET、EMAIL_*、WHATSAPP_LINK 等。

  • server.js — 應用程式進入點與所有路由

  • lib/ — 組別、規則、電郵

  • views/partials/ — 共用 partial
  • public/ — 靜態資源（含編譯後 CSS）
  • scripts/send-email-demo.js — 電郵 CLI 示範
